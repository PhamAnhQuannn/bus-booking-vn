# Ubiquitous Language / Domain Glossary

> Canonical definitions extracted from prisma/schema.prisma and codebase. Vietnamese equivalents included. Research date: 2026-06-17.

---

| Term | Vietnamese | Definition |
|------|-----------|------------|
| **Operator** | Nha Xe | Bus company registered on the platform. Follows a KYB approval lifecycle: `PENDING_REVIEW -> UNDER_REVIEW -> APPROVED \| REJECTED` (with `REJECTED -> PENDING_REVIEW` resubmit and `APPROVED <-> SUSPENDED` toggle). Only `APPROVED` operators are visible in search and bookable. Tax classification: `company` (exempt) or `individual_household` (4.5% PIT withholding via `tax_withheld` ledger entries). |
| **OperatorUser** | | Human account for the operator portal. Role: `admin` or `staff`. Login by username (format: `BRAND_ACRONYM-last4phone`), not by phone. Entirely distinct from Customer -- separate session model (`OperatorSession`), separate OTP model (`OperatorOtpAttempt`). Carries both `contactPhone` and `notificationPhone` (may be identical). |
| **Route** | Tuyen | Bidirectional bus line defined by free-text origin/destination strings plus optional `Place` foreign keys for canonical matching. Scoped to a single operator via `operatorId`. Can be soft-disabled by admin moderation (`moderatedAt` timestamp) or operator deactivation (`deactivatedAt`). Carries `durationMinutes` used for bus-overlap window calculations. |
| **Place** | | Global canonical registry of bookable place names. Supports alias merging (the `aliases` string array) and diacritics-normalized search via `unaccent_immutable ILIKE`. Each Place has a unique `slug` for URL-friendly identification. Not operator-scoped -- shared across the platform. |
| **Trip** | Chuyen | A specific scheduled departure on a Route by a Bus. Fixed price in VND (integer minor-unit), `departureAt` timestamp. Status enum: `scheduled \| departed \| completed \| cancelled`. The `salesClosed` boolean flag is orthogonal to status -- it prevents new holds and hides the trip from search. Optional `moderatedAt` for admin soft-hide. Optional `blockedSeats` integer reduces effective capacity. Can be linked to a paired return trip via `pairedTripId`. |
| **RecurringTripTemplate** | | Schedule definition for auto-generating trips. Specifies route, bus, price, `departureLocalTime` (HH:MM in Asia/Ho_Chi_Minh timezone), and `daysOfMask` (weekday bitmask). A daily cron job (`generateFromTemplate`) creates Trip rows for a 14-day rolling horizon. Deduplication via partial unique index prevents double-generation. Tracked in `RecurringGenerationLog`. |
| **Bus** | Xe | Physical vehicle belonging to an operator. Key fields: `capacity` (seat count), `licensePlate` (unique per operator), `busType` (`coach \| sleeper \| limousine`). Supports deactivation (`deactivatedAt`) and maintenance windows (`maintenanceStart`/`maintenanceEnd` on the Bus model, plus detailed `BusMaintenance` records). |
| **BusMaintenance** | | Scheduled maintenance window record for a Bus. Fields: `startAt`, `endAt`, optional `reason`. Prevents trip scheduling during the window. Trip search uses window-vs-window overlap logic to exclude buses with overlapping maintenance. |
| **Hold** | Dat Cho Tam | Time-bounded seat reservation with a 10-minute TTL (`HOLD_TTL_MINUTES=10`). Capacity checked atomically via pg advisory locks: phone-level lock first (cap guard), then trip-level lock (serialization). Created with `status='active'` and converted to a Booking upon payment initiation (`status='consumed'`). Other terminal states: `expired` (cron sweeper), `cancelled_trip` (trip cancellation cascade). Carries pickup selection fields. |
| **Booking** | Dat Ve | Confirmed ticket purchase. Created from a consumed Hold in `awaiting_payment` status. Transitions to `paid` after PSP webhook confirms successful payment. Carries a buyer snapshot (name, phone, email), ticket count, total in VND, payment method, and pickup details. Identified by two tokens: `bookingRef` (human-readable, for customer communication) and `confirmationToken` (URL-safe, for ticket page access). |
| **BookingRef** | | Human-readable booking reference. Format: `BB-YYYY-[0-9a-z]{4}-[0-9a-z]{4}` (base36 lowercase). Year prefix from booking creation date. Exported as `BOOKING_REF_REGEX` from `lib/booking/bookingRef.ts`. Used in payment webhook lookups and customer-facing displays. |
| **Ticket** | Ve | Not a separate database model. A Booking in `paid` or `completed` status IS the ticket. The PDF is generated asynchronously by a cron job and stored at `Booking.ticketPdfKey` in object storage. Customer views the ticket at `/booking/confirmation/[confirmationToken]`. |
| **Customer** | | Registered end-user identified by phone number. Supports soft-delete (`deletedAt`), data anonymization (`anonymizedAt`), and suspension (`suspendedAt`). Guest bookings (made without registration) are backfilled to the customer record on registration via phone match (`attachGuestBookingByPhone`). Optional email and password hash. |
| **Hold Cap / CONCURRENT_HOLD_CAP** | | Maximum number of simultaneous active holds allowed per phone number across all trips. Enforced via phone-level pg advisory lock (`pg_advisory_xact_lock(hashtext('hold-phone:' || phone))`). Prevents seat-squatting. |
| **PSP Window** | | `PSP_WINDOW_MINUTES = 20`. The grace period during which an `awaiting_payment` booking still occupies a seat in capacity calculations. Bookings older than this window are excluded from the seat count, allowing the hold/booking slot to be reclaimed even if the PSP has not yet sent a failure webhook. |
| **Payout** | Giai Ngan | Disbursement to an operator. Two creation paths: (1) auto-sweep per trip at `markCompleted` with `scheduledAt = completedAt + 1 day` (T+1 settlement), or (2) on-demand withdrawal by operator (`tripId=null`). Status lifecycle: `requested -> processing -> paid \| failed`. Failed payouts can be retried by admin (`retryPayout`). Carries `gross`, `platformFee`, `net`, and tax breakdowns (`taxVat`, `taxPit`, `taxTotal`). |
| **LedgerEntry** | | Double-entry accounting row. Append-only and immutable, enforced by PostgreSQL `BEFORE UPDATE/DELETE` trigger (`ledger_entry_immutable`). Positive amount = credit to operator, negative = debit. Types: `booking_credit`, `platform_fee`, `refund_debit`, `refund_out`, `payout_debit`, `payout_reversal`, `chargeback`, `adjustment`, `tax_withheld`. Each entry has a unique `sourceEventId` for idempotency. Balance is always derived from the sum of entries, never stored as a column. |
| **Platform Fee** | | Marketplace commission on each booking. Stored as `ratePpm` (parts-per-million) in `FeeConfig` -- e.g., `60000` = 6%. Effective-dated: new FeeConfig row for rate changes, never in-place edits. Supports global default (no `operatorId`) plus per-operator override. Computed via BigInt arithmetic in `calcPlatformFeeMinor` (`lib/ledger/feeConfig.ts`). |
| **Available Balance** | | Derived value, never stored. Computed as `settledEligible - paidOut`. Settlement-eligible entries require the linked trip's `completedAt + 1 day <= NOW()` (`SETTLEMENT_DELAY_DAYS=1`). Withdrawal requires available balance >= `MIN_WITHDRAW_THRESHOLD_VND` (100,000 VND). The `refund_out` entry type is deliberately excluded from balance (platform-float cash, already accounted for via paired `refund_debit`). |
| **OTP** | Ma OTP | 6-digit one-time password delivered via SMS (eSMS/SpeedSMS). Stored as salted SHA-256 hash in `OtpAttempt` (customer) or `OperatorOtpAttempt` (operator). TTL: 5 minutes. Rate limit: 3 failed verifications trigger a 15-minute lockout. The lockout is implemented as a sentinel by repurposing the same OTP row: `consumed=true`, `attemptCount >= 3`, `expiresAt = now + 15min`. New OTP sends supersede the prior active row via `ON CONFLICT (phone) WHERE consumed=false DO UPDATE`. |
| **otpProof JWT** | | Short-lived HS256 JSON Web Token (5-minute TTL) returned by `/api/auth/otp/verify` to prove phone ownership. Claims: `{ phone, jti, purpose: 'register' \| 'login' }`. Single-use: JTI consumed via Redis `SETNX`. Presented in the register/login request body. Listed in the logger's redact list to prevent leaking through structured logs. |
| **salesClosed** | | Boolean flag on Trip. When `true`: trip is excluded from search results and new hold creation fails (the conditional INSERT requires `salesClosed = false`). Automatically set to `true` by `markDeparted`. Can also be manually toggled by operator via `lib/trips/salesToggle.ts`. Orthogonal to Trip status -- a `scheduled` trip can have `salesClosed = true` (e.g., operator manually closes sales early). |
| **KYB (Know-Your-Business)** | | Operator document verification process. Document types: `business_license`, `identity`, `payout_account`. Files stored in object storage with keys tracked in `KybDocument`. Documents can be purged (`purgedAt`). The KYB review drives the `Operator.status` lifecycle from `PENDING_REVIEW` through `UNDER_REVIEW` to `APPROVED` or `REJECTED`. |
| **Charter** | Thue Xe | Group or private-hire request. Lead-generation only -- no payment rail on the platform. Customer submits origin, destinations (JSON), date range, passenger count, and vehicle type. Admin dispatches to an operator via direct assignment or public claim. Operator accepts or declines. Settlement handled entirely off-platform. Expiry sweeper auto-transitions stale requests. |
| **PickupKind** | | How the traveler boards the bus. Three values: `station` (Ben xe terminal -- the default depot), `point` (a named `OperatorPickupArea` defined by the operator), `custom` (free-text address, requires `pickupDetail` >= 5 chars, operator calls back to confirm). |
| **PickupPlaceKind** | | Classification of an `OperatorPickupArea`. Two values: `station` (Ben xe -- a bus terminal) or `pickup` (Don tan noi -- door-to-door pickup point). |
| **E-Invoice** | Hoa Don Dien Tu | GDT-compliant electronic invoice per Circular 32/2025/TT-BTC (replaces Circular 78/2021, effective 1 Jun 2025). Tracked in the `EInvoice` model with status lifecycle: `pending -> issued -> sent` (happy path), `-> failed`, or `-> cancelled`. Each invoice carries `invoiceNumber`, `issuedAt`, and a `vendorRef` for the MISA meInvoice integration. Referenced from Booking via `einvoiceRef`. |
| **AdminAuditLog** | | Append-only audit trail for all admin actions. Immutable via PostgreSQL `BEFORE UPDATE/DELETE` trigger. Fields: `actor`, `action`, `target`, `argsRedacted` (JSON with sensitive fields masked). Phone numbers masked to last 4 digits via `lib/audit/redactPhone.ts`. |
| **FunnelEvent** | | Anonymous conversion funnel instrumentation. Event steps: `search_performed`, `hold_created`, `payment_initiated`, `booking_paid`. Keyed by `sessionId` (anonymous), optionally linked to `tripId` and `bookingId`. Carries a `context` JSON field. No PII stored. |
| **ConsentRecord** | | Immutable record of user consent captured at booking initiation. Types: `no_refund` (acknowledgement of no-refund policy) and `pii_storage` (consent for personal data storage). Carries `version` and `consentedAt`. Protected by the same immutability trigger pattern as LedgerEntry. |
| **ContactStatus** | | Booking-level field tracking operator-to-customer contact progress for custom pickup coordination. Values: `pending`, `reached`, `no_answer`, `callback`. |
| **StoredObject** | | Object storage metadata record. Tracks `key` (unique storage path), `contentType`, `sizeBytes`, `purpose`, and `uploadedBy`. Used by KYB document uploads and ticket PDF storage. |
| **FeatureFlag** | | Runtime feature gate. Simple `key`/`enabled`/`value` model. `updatedBy` tracks who last changed the flag. Used for gradual rollouts and operational toggles. |
| **ApplicationRef** | | Unique reference string generated for each operator registration, used for tracking the onboarding application through the approval pipeline. |
