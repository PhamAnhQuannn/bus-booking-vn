Fix: extracted the default-range computation to a module-scope `getDefaultDateRange()` helper outside the component function; the render body calls the helper. The helper is still impure (calls `Date.now()` internally) but is now invoked at the well-defined boundary of "compute defaults at request time" rather than embedded mid-render. Rule: server-component render bodies must not call `Date.now()`, `Math.random()`, `crypto.randomUUID()`, or any other non-deterministic API. Compute time-derived defaults in a module-scope helper, in a route handler before passing as props, or in a client component (`'use client'`) where re-render semantics handle the non-purity correctly. Greppable smell: `Date.now()` / `Math.random()` inside `export default async function <Name>Page(...)` body in any `app/**/page.tsx` is a `react-hooks/purity` violation waiting to fire.
