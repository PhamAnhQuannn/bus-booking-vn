## Findings

### ❌ Blockers (real bugs — fix before any further visual work)

1. **Sans font never binds → everything renders in browser-default serif.**
   `app/globals.css` `@layer base` sets `html { @apply font-sans }` →
   `font-family: var(--font-sans)`. But `@theme inline` (line 10) defines
   `--font-sans: var(--font-sans)` — **self-referential/undefined**. `app/layout.tsx`
   loads Geist as `--font-geist-sans`, which is never mapped to `--font-sans`. Result:
   all body/heading text falls back to a serif system font (visible in every screenshot —
   the "Times-y" headings). Mono is fine (`--font-mono: var(--font-geist-mono)`).
   **Fix (1 line):** `--font-sans: var(--font-geist-sans);` in `@theme`. This alone removes
   much of the "undesigned" feel.

### ⚠️ Visual / product-chrome gaps (the "stock shadcn starter" tells)

2. **No app shell — no header, logo, nav, or footer.** Every page is bare content floating
   on white. No brand presence, no wayfinding, no global actions (login/account link). This
   is the #1 "not a product" signal. Customer side has zero chrome; operator side has the
   `(console)` sidebar but the login/entry is bare.

3. **Content floats in a sea of whitespace on desktop.** Forms + results sit in a centered
   `max-w-md`/`max-w-2xl` column with huge empty margins (see search-results: one card
   mid-canvas). Wireframes intentionally kept mobile-first centering, but desktop reads
   empty. Needs a layout container + anchored top alignment / supporting structure.

4. **Flat, no depth or sectioning.** Single `shadow-sm` cards on a pure-white page; nothing
   separates regions. No background tint, no elevation hierarchy, no hero band. Everything
   is one z-plane.

5. **Orange reads "tacked on," not integrated.** Brand color appears only as accents (price,
   CTA, link, the "vé" word). No brand surface — no orange header bar, hero, or filled
   brand moments. Color ≠ identity yet.

6. **Zero iconography.** `lucide-react` is installed but unused in these views. Travel/booking
   UI expects route arrows, clock, seat/bus, phone, payment glyphs. All-text cards feel bare.

7. **Home is a bare title + form — no value prop, trust, or imagery.** No headline selling
   "book in 30s," no payment-method/trust row, no operator/route imagery. First impression
   is a naked form.

8. **Search results lack list affordances.** No result count, no sort/filter controls, no
   empty-state illustration; a single result looks lonely. (`/search` page.)

### ✅ Works (keep)

- **Consistency:** all surfaces now share Card / Input / Button / Badge primitives + tokens
  (post-migration). No inline styles. This is a strong foundation.
- **Orange brand applied** coherently to CTAs, price, links, focus rings; surfaces stay clean.
- **Legibility & spacing within cards** is good; forms have proper labels + focus states (a11y).
- **Price/route hierarchy** on the result card is clear; status Badges are semantic.
- **Operator console** has a real sidebar shell (the most "product-like" surface today).

## Readiness verdict

**Status: 🟡 yellow.** The UI is *consistent and on-brand-color* but does not yet *look like
a product* — it reads as a shadcn starter because of (a) the serif-font bug, and (b) missing
product chrome (header/nav, hero, depth, icons, brand surfaces). None of this is structural;
it's a focused "product skin" pass on a sound foundation.

- **Fix immediately (bug):** finding #1 (font binding) — 1 line, outsized payoff.
- **Then (visual identity):** feed findings #2–#7 into `/visual-mood-board` (direction) and a
  follow-up implementation task (app shell + hero + icons + depth).
